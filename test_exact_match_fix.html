Purpose: Verify that changing from includes() to exact match (===) fixes the column visibility issue.

Problem Analysis

The issue was that header.toLowerCase().includes(col.toLowerCase()) was doing partial matches:

  • 'a' matched columns like 'account_mgr', 'final_amt', 'date_received'
  • 'c' matched columns like 'client', 'pic', 'decision'
  • 'r' matched columns like 'rev', 'project_name', 'date_received'
  • 'u' matched columns like 'uid', 'solutions'
  • 'd' matched columns like 'decision', 'date_received'

This meant almost every column was being hidden except for columns like 'bom' that don't contain these letters!
