Decision Field Options Fix Verification

Issue Summary

There was an issue where only the "decision" field was showing up as a dropdown but had incorrect options ("Pending", "Approved", "Declined") instead of the expected options ("", "GO", "DECLINE") from the getFieldOptions function.

The A, C, R, U, D, solutions, sol_particulars, industries, ind_particulars fields were also not showing up as dropdowns.

Applied Fixes

  1. Fixed hardcoded dropdown options in index.html Create form for "decision" field
  2. Added clearing of existing options in dropdown creation for both Edit and Create modal functions
  3. Updated the filter logic for 'no_decision' to check for empty decision instead of 'pending'
  4. Simplified declined check to only look for 'decline'
  5. Added 'a', 'c', 'r', 'u', 'd', and 'submitted' to the DROPDOWN_FIELDS constant (critical fix)

Testing Steps

Verification pending. Please follow the testing steps.

  1. Click "Create New Opportunity" button
  2. Verify that the "Decision" dropdown shows options: "--Select--", "GO", and "DECLINE"

  4. Verify that solutions, sol_particulars, industries, and ind_particulars also show up as dropdowns

  2. Verify that the "Decision" dropdown shows options: "--Select--", "GO", and "DECLINE"

  4. Verify that solutions, sol_particulars, industries, and ind_particulars also show up as dropdowns
